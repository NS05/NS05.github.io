Hexo-Theme Next configure and prettify

NS05 · 2019-11-30 · 次阅读

Hexo 主题种类繁多，除了网上搜索之外（如github中Hexo主题），你当然可以自己创建自己的主题，参考Hexo主题官方文档，但显然，非Web前端工程师很难自创主题，并配置完善美化良好。当然，我也仅仅是个通信工程专业的Tyro，对简雅风格喜欢的我选择了 Next ，只可惜配置过程并不是很容易，这篇博客即是我对 Next 主题调整的过程的记述。<本文将一直更新>

Next 基本配置

Next 主题下载

Next 主题版本有许多，但请注意，Hexo 如果更新至 4.0.0 ，低版本的 Next 主题“上一页”“下一页”的按钮将会渲染失败，呈现出裸露的代码，据说需要将 Next 升级至 7.4.2 以上才可，故接下来的配置是对 Next 7.4.2 进行配置。首先你要有 Next 主题文件包，Don’t worry, it’s free. 现在就下载 >>> 点击这里

Source code (zip) 即是 Next 7.4.2 主题文件包，点击下载

因习惯，将下载下来的文件的文件名由 hexo-theme-next-7.4.2 改为 next ，将其放置到 MyBlog（你博客文件夹的名称，下文将均写为MyBlog） >>> themes 中，这样，准备工作就完成了

Next 主题配置

下面进行主题的基本配置

Hexo 配置

打开 MyBlog >>> _config.yml 文件

PS：推荐使用 Sublime Text 打开，macOS 可通过右键该文件 >>> 打开方式 >>> Sublime Text >>> 全部更改… 来设置其为默认的应用程序

修改中文注释的配置

# Hexo Configuration
## Docs: https://hexo.io/docs/configuration.html
## Source: https://github.com/hexojs/hexo/

# Site
title: Hexo       #你的博客标题
subtitle: 'xxx'   #你的博客副标题，可不设置
description: ''   #网站的描述，可不设置
keywords:         #网站关键词，可不设置
author: author        #作者
language: zh-CN        #语言(MyBlog >>> themes >>> next >>> languages 中的语言均可选择)
timezone: ''            #时区，默认使用电脑时区，可不设置

# URL
## If your site is put in a subdirectory, set url as 'http://yoursite.com/child' and root as '/child/'
url: https://用户名.github.io.com         #你的博客网址
root: /
permalink: :year/:month/:day/:title/
permalink_defaults:
pretty_urls:
  trailing_index: true # Set to false to remove trailing index.html from permalinks

# Directory
source_dir: source
public_dir: public
tag_dir: tags
archive_dir: archives
category_dir: categories
code_dir: downloads/code
i18n_dir: :lang
skip_render:

# Writing
new_post_name: :title.md # File name of new posts
default_layout: post
titlecase: false # Transform title into titlecase
external_link:
  enable: true # Open external links in new tab
  field: site # Apply to the whole site
  exclude: ''
filename_case: 0
render_drafts: false
post_asset_folder: true
relative_link: false
future: true
highlight:
  enable: true
  line_number: true
  auto_detect: false
  tab_replace: ''

# Home page setting
# path: Root path for your blogs index page. (default = '')
# per_page: Posts displayed per page. (0 = disable pagination)
# order_by: Posts order. (Order by date descending by default)
index_generator:
  path: ''
  per_page: 10
  order_by: -date

# Category & Tag
default_category: uncategorized
category_map:
tag_map:

# Metadata elements
## https://developer.mozilla.org/en-US/docs/Web/HTML/Element/meta
meta_generator: true

# Date / Time format
## Hexo uses Moment.js to parse and display date
## You can customize the date format as defined in
## http://momentjs.com/docs/#/displaying/format/
date_format: YYYY-MM-DD
time_format: HH:mm:ss
## Use post's date for updated date unless set in front-matter
use_date_for_updated: false

# Pagination
## Set per_page to 0 to disable pagination
per_page: 10
pagination_dir: page

# Include / Exclude file(s)
## include:/exclude: options only apply to the 'source/' folder
include:
exclude:
ignore:

# Extensions
## Plugins: https://hexo.io/plugins/
## Themes: https://hexo.io/themes/
theme: next                           #主题(MyBlog >>> themes 中主题都可选，此处选 next )
# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  repository: https://github.com/NS05/NS05.github.io.git
  branch: master

官方文档更加详细，了解更多

Next 配置

配置主题

尽管 Next 已经是一个主题，但开发者又为我们配备了4种样式

打开 MyBlog >>> themes >>> next >>> _config.yml 文件

找到 Schemes

# Schemes
#scheme: Muse
#scheme: Mist
scheme: Pisces  #本博客使用该样式，你当然可以选择其他样式
#scheme: Gemini
配置目录

下图所示位置

打开 MyBlog >>> themes >>> next >>> _config.yml 文件

找到 menu 暂且修改中文注释的位置

menu:
  home: /|| home
  categories: /categories/|| th          #分类
  tags: /tags/|| tags                    #标签
  archives: /archives/ || archive        #归档
  #schedule: /schedule/ || calendar
  #sitemap: /sitemap.xml || sitemap
  #commonweal: /404/ || heartbeat
  about: /about/|| user                  #关于

PS: 本来空格会导致 Cannot GET/xxx 错误，然而 Hexo 4.0.0 与 Next 7.4.2 似乎没有这种问题，如：

categories: /categories/ || th 会导致错误 （注意空格）

categories: /categories/|| th 不会导致错误

但既然 Hexo 4.0.0 与 Next 7.4.2 不会产生这种问题，就不用顾虑，只要你需要，就可以将前面 # 号去掉，来使用。

接下来， 右键 MyBlog >>> 新建位于文件夹位置的终端窗口 ，输入

sudo hexo new page categories

这样我们发现，在 MyBlog >>> source 出现了 categories 文件夹，那么我们打开博客页面，点击分类，显示的即是这个文件夹页面。打开这个文件夹，打开里面的 index.md 文件，配置成如下格式，注意，<>是注释，另请注意“ : ”后的空格。

---
title: categories           <!---标题 可修改--->
date: 2019-11-29 14:17:48   <!---创建日期--->
type: "categories"          
layout: "categories"
---

这样，分类页面就做好啦，同样的道理，除了归档不需操作以外，其余页面均需这样操作。

标签页面：

右键 MyBlog >>> 新建位于文件夹位置的终端窗口 ，输入

sudo hexo new page tags

打开 MyBlog >>> source >>> tags >>> index.md 修改为

---
title: tags
date: 2019-11-29 14:46:08
type: "tags"
layout: "tags"
---

关于页面

右键 MyBlog >>> 新建位于文件夹位置的终端窗口 ，输入

sudo hexo new page about

打开 MyBlog >>> source >>> about >>> index.md 修改为

---
title: About Me
date: 2019-11-29 14:45:55
type: "about"
layout: "about"
---

在关于页面，你可以添加一些东西，比如

---
title: About Me
date: 2019-11-29 14:45:55
type: "about"
layout: "about"
comments: true
---

<center>
QQ:1157155009</br>
感谢你的关注～</br>
</center>

语言是 Markdown 可参考本站博客 Typora and Markdown

至此，基本的配置就完成了，可 右键MyBlog >>> 新建位于文件夹位置的终端窗口 ，逐次输入

sudo hexo clean
sudo hexo g
sudo hexo s

然后在本地查看，觉得无误后，Ctrl+C ，输入

sudo hexo d

就上传至网页上了，以后每次改动，都这样操作，下文不再赘述
